# Goal

Make the progress bar render on a **single line** instead of repeating with the long filename appended each time.

# Scope / Non-goals

- **In scope:** Fix the `ProgressStyle` template and finish behavior in `progress.rs`
- **Out of scope:** Changing the text-mode renderer, changing `merger.rs` logic, adding new features

# Current State And Constraints

The current `ProgressStyle` template in [src/progress.rs](src/progress.rs:126):

```rust
"[{elapsed_precise}] {bar:30.cyan/blue} {pos}/{len} ({per_sec}) {msg}"
```

The `{msg}` placeholder receives the full filename (e.g. `✓ 【小超梦】与高手鳄鱼相互单杀... (506ms)`) via `bar.set_message()`. When `bar.inc(1)` is called, `indicatif` redraws the line — but because the message is very long, the line wraps and appears to print multiple times. The user sees 3 lines:

1. `[00:00:00] ------------------------------ 0/1 (0.0 files/s)` — initial state (empty msg before first record)
2. `[00:00:00] ============================== 1/1 (2.0 files/s) ✓ 【小超梦】...` — after `set_message()` + `inc(1)`
3. `[00:00:00] ============================== 1/1 (1.9 files/s) ✓ 【小超梦】... (506ms)` — after `finish()` which uses `finish()` keeping the message

With rayon parallelism and fast operations, the redraw is visible before the terminal clears the previous line.

# Design

**Remove `{msg}` from the progress bar template.** This is the simplest and most effective fix:

1. Change the template to: `"[{elapsed_precise}] {bar:30.cyan/blue} {pos}/{len} ({per_sec})"`
2. Change `bar.finish()` → `bar.finish_and_clear()` so the progress bar disappears after completion, leaving a clean terminal for the summary report that `MergeSummary::print_report()` already prints.
3. The `set_message()` and `format_record()` calls in the `Renderer::Bar` branch become no-ops visually (the message is set but never displayed in the template). We keep them in code to maintain the API — they don't cause any output harm.

This approach:
- Guarantees a single-line progress bar (no long filename wrapping)
- The summary report already provides per-file results when there are failures
- No new dependencies or complex logic needed

# Risks And Compatibility

- **Low risk:** The `{msg}` was only decorative in the progress bar; `MergeSummary::print_report()` already shows all results after completion.
- If a user relied on seeing per-file progress in real-time within the bar, they can still use `--verbose` flag or check the summary.
- `finish_and_clear()` is available in indicatif 0.17.x (current dependency is 0.17.8).

# Rollback

Revert the two changes:
1. Add `{msg}` back to the template string
2. Change `finish_and_clear()` back to `finish()`
