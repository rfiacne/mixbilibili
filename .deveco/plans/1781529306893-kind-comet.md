# mixbilibili 项目优化计划（12项）

## Goal

对 mixbilibili 项目进行代码质量、性能、功能完整性和可维护性的多维度优化，使其更健壮、更高效、更易扩展。

## Scope / Non-goals

**包含（12项）：**
1. 修复 scanner.rs 硬编码英文错误信息
2. MergeState Vec→HashSet 性能优化
3. 修复 progress.rs Text 模式 Clone 计数器不共享
4. Linux 包管理器扩展
5. i18n HashMap 重构
6. rayon 中断支持
7. 递归扫描子目录选项
8. CI 添加 macOS 构建
9. 集成测试增强
10. install.sh 增强
11. 删除源文件并行→串行
12. dry_run 代码路径清理

**不包含：**
- 状态文件迁移到平台标准目录（原#8，用户明确排除）
- 重大架构重构（如 async 化、引入 tokio）
- UI/UX 大改（如 TUI 界面）

## Current State And Constraints

- **语言**: Rust 2021 edition, 当前版本 0.6.2
- **依赖**: clap 4.5, rayon 1.10, colored 2.1, anyhow 1.0, indicatif 0.17, thiserror 1.0, serde 1.0, ctrlc 3.4, which 6.0
- **模块**: cli, ffmpeg, i18n, merger, progress, scanner, state
- **测试**: 约 60+ 单元测试 + 4 集成测试，CI 在 ubuntu-latest 运行
- **发布**: GitHub Actions 自动构建 Linux(x86_64, aarch64) + Windows(x86_64)

### #1 修复 scanner.rs 中硬编码英文错误信息（High）

**问题**: `scan_directory()` 使用 `.context("Failed to read directory")` 和 `.context("Failed to read directory entry")` 硬编码英文，与项目 i18n 体系不一致。

**方案**:
1. 在 [src/i18n.rs](src/i18n.rs) 添加翻译键：
   - `"failed_read_dir"` — 读取目录失败
   - `"failed_read_entry"` — 读取目录条目失败
2. 在 [src/scanner.rs](src/scanner.rs) 替换：
   - `.context("Failed to read directory")` → `.context(t("failed_read_dir").into_owned())`
   - `.context("Failed to read directory entry")` → `.context(t("failed_read_entry").into_owned())`

**涉及文件**: `src/i18n.rs`, `src/scanner.rs`

### #2 MergeState Vec→HashSet（High）

**问题**: `completed`、`failed`、`pending` 均为 `Vec<String>`，`mark_completed`、`mark_failed`、`is_completed`、`add_pending` 中的 `.iter().any()` 和 `.retain()` 均为 O(n) 操作。

**方案**:
1. 将 `MergeState` 内部存储改为 `HashSet<String>`：
```rust
pub struct MergeState {
    pub source_dir: String,
    pub output_dir: String,
    pub format: String,
    #[serde(serialize_with = "set_as_vec", deserialize_with = "vec_as_set")]
    pub completed: HashSet<String>,
    #[serde(serialize_with = "set_as_vec", deserialize_with = "vec_as_set")]
    pub failed: HashSet<String>,
    #[serde(serialize_with = "set_as_vec", deserialize_with = "vec_as_set")]
    pub pending: HashSet<String>,
}
```
2. 添加 serde 辅助函数，确保 JSON 格式仍为数组（向后兼容旧状态文件）：
```rust
fn set_as_vec<S: serde::Serializer>(set: &HashSet<String>, s: S) -> Result<S::Ok, S::Error> {
    let vec: Vec<&String> = set.iter().collect();
    vec.serialize(s)
}
fn vec_as_set<'de, D: serde::Deserializer<'de>>(d: D) -> Result<HashSet<String>, D::Error> {
    let vec: Vec<String> = Vec::deserialize(d)?;
    Ok(vec.into_iter().collect())
}
```
3. 简化方法实现：
   - `is_completed` → `self.completed.contains(stem)` (O(1))
   - `mark_completed` → `self.pending.remove(stem); self.completed.insert(stem.to_string())`
   - `mark_failed` → `self.pending.remove(stem); self.failed.insert(stem.to_string())`
   - `add_pending` → 单次 `contains` 检查 + `insert`

4. 更新 [src/main.rs](src/main.rs) 中 `apply_results` 函数，适配 `HashSet` 接口（`contains` 替代 `iter().any()`）。

5. 更新 [src/main.rs](src/main.rs) 中 `finalize` 函数，`pending.iter().map()` 需适配 `HashSet` 迭代。

**涉及文件**: `src/state.rs`, `src/main.rs`

### #3 修复 progress.rs Text 模式 Clone 计数器不共享（High）

**问题**: `MergeProgress::clone()` 在 `Renderer::Text` 分支新建 `AtomicUsize`，导致各 clone 之间的 `completed` 计数器独立。在 rayon 并行环境中，每个线程持有独立 clone，`completed` 不会同步递增，导致文本模式进度计数失效。

**方案**: 将 `Renderer::Text` 中的 `completed: AtomicUsize` 改为 `Arc<AtomicUsize>`，Clone 时共享同一个 `Arc`：

```rust
Text {
    total: usize,
    completed: Arc<AtomicUsize>,
}

// Clone 实现
Renderer::Text { total, completed } => Renderer::Text {
    total: *total,
    completed: Arc::clone(completed), // 共享同一个计数器
}
```

`new_text()` 也相应修改：
```rust
fn new_text(total: usize) -> Self {
    Self::Text {
        total,
        completed: Arc::new(AtomicUsize::new(0)),
    }
}
```

**涉及文件**: `src/progress.rs`

### #5 i18n.rs translate() 使用 O(n) 线性匹配，改用 HashMap（Medium）

**问题**: `translate()` 使用巨大的 `match` 语句，每次调用都要从头匹配，时间复杂度 O(n)。

**方案**: 使用 `OnceLock<HashMap>` 在首次调用时初始化，无需引入新依赖：

1. 定义静态翻译表为常量数组：
```rust
const TRANSLATIONS: &[((Lang, &str), &str)] = &[
    ((Lang::Cn, "error_prefix"), "错误："),
    ((Lang::En, "error_prefix"), "Error:"),
    // ... 所有翻译条目
];
```

2. 使用 `OnceLock` 缓存 HashMap：
```rust
static TRANSLATION_MAP: OnceLock<HashMap<(Lang, &str), &str>> = OnceLock::new();

fn translation_map() -> &'static HashMap<(Lang, &str), &str> {
    TRANSLATION_MAP.get_or_init(|| {
        TRANSLATIONS.iter().map(|&(k, v)| (k, v)).collect()
    })
}
```

3. `translate()` 改为查表：
```rust
fn translate(lang: Lang, key: &str) -> Cow<'static, str> {
    translation_map()
        .get(&(lang, key))
        .map(|s| Cow::Borrowed(*s))
        .unwrap_or_else(|| Cow::Owned(key.to_string()))
}
```

**注意**: `Lang` 需要派生 `Hash` 和 `Eq`（当前已有 `PartialEq, Eq`，需加 `Hash`）。`(Lang, &str)` 作为 HashMap 键需要 `Hash` 实现。

**涉及文件**: `src/i18n.rs`

### #6 merger.rs 中 Ctrl+C 中断无法停止 rayon 并行任务（Medium）

**问题**: `execute_merges()` 使用 `rayon::par_iter()`，即使 `INTERRUPTED` 标志被设置，正在运行的 rayon 任务池仍会处理完所有分配的任务。

**方案**:
1. 在 [src/main.rs](src/main.rs) 中将 `INTERRUPTED` 改为 `pub`：
```rust
pub static INTERRUPTED: AtomicBool = AtomicBool::new(false);
```

2. 在 [src/merger.rs](src/merger.rs) 的 `merge_pair()` 函数开头添加中断检查：
```rust
pub fn merge_pair(...) -> MergeResult {
    // 检查中断标志
    if crate::INTERRUPTED.load(Ordering::SeqCst) {
        return MergeResult {
            pair_index,
            pair_name: pair.stem.clone(),
            success: false,
            error: Some("Interrupted".to_string()),
            duration: Duration::ZERO,
        };
    }
    // ... 原有逻辑
}
```

3. 在 `execute_merges()` 的 `par_iter().map()` 内，每个任务开始前也检查中断，已中断的任务直接跳过不启动 ffmpeg。

**涉及文件**: `src/main.rs`, `src/merger.rs`

### #7 缺少递归扫描子目录选项（Medium）

**问题**: Bilibili 下载工具通常按番剧名或合集创建子目录，当前 `scan_directory()` 只扫描一层。

**方案**:
1. 在 [src/cli.rs](src/cli.rs) 添加 `--recursive / -R` 参数：
```rust
.arg(
    Arg::new("recursive")
        .short('R')
        .long("recursive")
        .help(t("cli_recursive"))
        .action(ArgAction::SetTrue),
)
```

2. `Args` 结构体添加 `pub recursive: bool` 字段。

3. 在 [src/scanner.rs](src/scanner.rs) 修改 `scan_directory` 签名和行为：
```rust
pub fn scan_directory(source_dir: &Path, recursive: bool) -> Result<ScanResult>
```
当 `recursive=true` 时，遇到子目录则递归调用自身；当 `recursive=false` 时保持当前行为（跳过子目录）。

4. 在 [src/i18n.rs](src/i18n.rs) 添加翻译键 `"cli_recursive"`。

5. 在 [src/main.rs](src/main.rs) 的 `scan_and_filter` 中传递 `args.recursive`。

**涉及文件**: `src/cli.rs`, `src/scanner.rs`, `src/i18n.rs`, `src/main.rs`

### #12 删除源文件并行→串行（Low）

**问题**: [src/merger.rs](src/merger.rs) 第 432-446 行使用 `par_iter()` 并行删除源文件。文件删除是元数据操作，磁盘 IO 天然顺序化，并行删除收益极低且增加 IO 调度压力。

**方案**: 将 `par_iter()` 改为 `iter()`：

```rust
// 改前
let deletion_failures: usize = results
    .par_iter()
    .filter(|r| r.success)
    .map(|result| { ... })
    .sum();

// 改后
let deletion_failures: usize = results
    .iter()
    .filter(|r| r.success)
    .map(|result| { ... })
    .sum();
```

同时移除该函数作用域内不再需要的 `use rayon::prelude::*`（如果该文件其他地方仍使用 rayon 则保留）。

**涉及文件**: `src/merger.rs`

### #13 dry_run 代码路径清理（Low）

**问题**: `main.rs` 的 `run()` 在 `args.dry_run` 时已单独输出预览（第 182-209 行），但 `execute_merges` 仍接收 `dry_run` 参数并在 `merge_pair` 中有独立的 dry-run 分支。两条路径存在逻辑冗余。

**方案**: 
1. 确认 `run()` 的 dry-run 分支已完全覆盖预览需求，直接 `return Ok(())`（当前代码第 208 行已如此）
2. 在 `execute_merges` 和 `merge_pair` 中移除 `dry_run` 参数，简化函数签名
3. 移除 `do_dry_run` 辅助函数
4. 更新所有调用点（`main.rs` 的 `execute()` 函数不再传 `dry_run`）

**涉及文件**: `src/merger.rs`, `src/main.rs`

## Risks And Compatibility

- **serde 兼容性**: HashSet 序列化需使用自定义 `serialize_with`/`deserialize_with` 确保旧状态文件（JSON 数组格式）仍可读取
- **rayon 中断**: 添加中断检查点增加每次 `merge_pair` 一次 `AtomicBool::load`（纳秒级，可忽略）
- **i18n HashMap**: 使用 `OnceLock` 初始化，首次调用有一次性开销，后续无影响；需确保 `Lang` 派生 `Hash`
- **递归扫描**: `--recursive` 为可选标志，默认不启用，完全向后兼容
- **macOS CI**: macOS runner 计费约为 Linux 的 10 倍，需注意 CI 成本
- **dry_run 清理**: 移除 `merge_pair` 的 dry_run 参数后需确认所有调用点更新
- **集成测试**: 依赖系统 ffmpeg，需在无 ffmpeg 环境中 graceful skip

## Rollback

每个优化项独立，可单独回滚：
- **代码变更**: `git revert <commit>` 回滚特定变更
- **state.rs HashSet**: 恢复 Vec 版本，旧状态文件格式不受影响（自定义 serde 保持 JSON 数组兼容）
- **i18n HashMap**: 恢复 match 版本，无外部依赖变更
- **递归扫描**: 移除 `--recursive` 参数和 scanner 递归逻辑
- **CI 变更**: 移除 macOS matrix 条目
- **dry_run 清理**: 恢复 `merge_pair` 的 `dry_run` 参数
- **并行→串行删除**: 改回 `par_iter()`
